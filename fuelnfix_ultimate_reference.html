System Architecture Reference

Welcome to the ultimate technical reference for FuelnFix. This document details every file, function, and logical flow within the application.

Tech Stack

  • Frontend: React.js, Vite, TailwindCSS, Framer Motion, Socket.io-client, Google Maps API
  • Backend: Node.js, Express.js, MongoDB (Mongoose), Socket.io, Firebase Admin (Auth)
  • Services: Cashfree (Payments), OSRM (Routing/Simulation), Google Geocoding

Server Core Files

server/index.js Entry Point

The main application entry point. Initializes Express, MongoDB connection, and Socket.io server.

DB Connection

Connects to MongoDB Atlas using `process.env.MONGO_URI`.

Socket.io Logic

Handles real-time events:

  • join_room: Users/Providers join their own ID rooms for private comms.
  • request_help: Emitted by User, broadcasted to Providers.
  • accept_request: Provider accepts, notifies User.
  • provider_location_update: Relays live GPS coords from Provider to User.
  • request_status_update: Updates phases (Arrived, Completed).

Database Models (Mongoose)

The application uses MongoDB with Mongoose for data modeling. The schema handles users (both customers and providers), service requests, reviews, and configuration.

server/models/User.js Schema

Stores both Customer and Provider profiles. We use a single collection with a `role` discriminator to simplify auth.

const UserSchema = new mongoose.Schema({
    name: { type: String, required: true },
    email: { type: String, required: true, unique: true },
    role: { type: String, default: 'user' }, // 'user', 'provider', 'admin'
    
    // Provider Specific
    shopName: { type: String },
    providerCategory: [{ type: String, enum: ['Mechanic', 'Fuel Delivery', 'EV Support'] }],
    location: { 
        type: { type: String, default: 'Point' }, 
        coordinates: [Number] // [Lng, Lat] - Shop Location
    },
    isVerified: { type: Boolean, default: false },
    pendingUpdate: { ... } // Staging area for profile changes requiring admin approval
});
server/models/Request.js Schema

The core business entity. Represents a service job from creation to completion and payment.

const RequestSchema = new mongoose.Schema({
    customer: { type: ObjectId, ref: 'User' },
    provider: { type: ObjectId, ref: 'User' },
    status: { type: String, enum: ['Pending', 'Accepted', 'Arrived', 'In Progress', 'Completed', 'Cancelled'] },
    
    pricing: {
        baseFee: Number,
        distanceFee: Number,
        materialCost: Number, // Added after job completion
        totalAmount: Number
    },
    
    serviceOtp: String, // 4-digit code for start verification
    otpVerified: Boolean,
    
    bill: [{ name: String, cost: Number }], // Itemized bill
    paymentStatus: { type: String, enum: ['pending', 'success', 'failed'] }
});
server/models/ServiceRate.js Schema

Configurable pricing model. Admins can update base prices and per-km rates for any service without code changes.

Fields
  • serviceName: Unique ID (e.g., 'Flat Tire', 'Petrol')
  • basePrice: Minimum call-out fee.
  • pricePerKm: Distance multiplier (Roadside Assist).
  • pricePerLitre: Fuel cost (Fuel Delivery only).
server/models/Review.js Schema

Bidirectional review system. Users review Providers, and Providers review Users.

const ReviewSchema = new mongoose.Schema({
    request: { type: ObjectId, ref: 'Request' },
    reviewer: { type: ObjectId, ref: 'User' },
    reviewee: { type: ObjectId, ref: 'User' },
    rating: { type: Number, min: 1, max: 5 },
    comment: String
});

stat.getAverageRating(userId): Aggregation pipeline to calc avg score.

Server Logic & Controllers

The core business logic is encapsulated in Controllers, keeping Routes clean. Key workflows include the Request Lifecycle, Payments, and Real-time updates.

server/controllers/requestController.js Controller

Handles the entire lifecycle of a service request.

searchProviders(req, res)

Complex geospatial query to find eligible providers.

  • POST /api/requests/search
  • Logic:
    1. Calculates a "Bundle Price" based on selected services from ServiceRate collection.
    2. Queries `User` collection for providers who are: isOnline: true, isVerified: true, and within 50km.
    3. Returns list with estimated cost, distance, and provider details.
createRequest(req, res)

Initializes a new job. Emits new_request socket event to the specific provider.

verifyServiceOtp(req, res)

Critical security step. Provider must enter the 4-digit OTP generated upon acceptance to start work.

  • POST /api/requests/verify-otp
  • Validation: Checks if OTP matches database.
  • Side Effect: Sets status to 'In Progress', emits otp_verified to User via Socket.io.
sendBill(req, res)

Generates the final bill. Handles dynamic logic for Fuel vs Mechanic.

  • Fuel: Calculates cost based on Litres * Rate.
  • Mechanic: Sums up itemized parts/labor list.
  • Output: Emits bill_received socket event to User with payment prompt.
server/routes/paymentRoutes.js Route

Integration with Cashfree Payment Gateway.

/create-order

Initiates a payment session with Cashfree.

const request = {
    order_amount: orderAmount,
    order_currency: "INR",
    customer_details: { ... },
    order_note: "FuelnFix - Roadside Assistance"
};
const response = await cashfree.PGCreateOrder(request);
/verify-payment

Backend verification of the payment status after frontend success callback.

server/routes/reviewRoutes.js Route

Handles rating submission and average calculation.

  • Submit: Updates the specific `Review` document AND recalculates the `User.averageRating` aggregate immediately.
  • Get: Fetches pagination-supported reviews for a profile.

Client Components

Reusable UI components that power the frontend experience.

client/src/components/RequestHelpWizard.jsx Component

The core booking flow. A complex multi-step modal that guides the user from service selection to provider booking.

Steps
  1. Select Services: Multiselect grid.
  2. Select Vehicle: Choose from user's registered vehicles.
  3. Add Photo: Upload problem image to Firebase Storage.
  4. Confirm Location: Fullscreen map to pin exact GPS coords.
  5. Select Provider: Map view of nearby providers. Clicking a marker opens the "Best Offer" bottom sheet.
handleBookProvider()

Compiles all wizard state (Vehicle, Photo URL, Location, Selected Provider) and calls /api/request/create.

client/src/components/LiveChat.jsx Component

Real-time floating chat widget using Socket.io.

  • Props: socket, roomId (Request ID).
  • Features:
    • Minimizable floating bubble.
    • Unread message badge with bounce animation.
    • Auto-scroll to bottom on new message.
client/src/components/GoogleMaps/MapComponent.jsx Wrapper

Encapsulates Google Maps configuration.

  • Supports Dark Mode (Uber-style custom JSON theme).
  • Handles Geolocation (Blue Dot).
  • Accepts children for rendering markers (Provider icons).

Deployment & Configuration

Environment Variables (Server)

  • MONGO_URI: MongoDB connection string.
  • CASHFREE_APP_ID: Payment Gateway ID.
  • CASHFREE_SECRET_KEY: Payment Gateway Secret.
  • FIREBASE_SERVICE_ACCOUNT: JSON string for Firebase Admin SDK.
