Server Entry Point

server/index.js

The server/index.js file is the backend's entry point. It has three main responsibilities: initializing the Express app, connecting to MongoDB, and setting up the Socket.IO server.

Real-Time Location Logic

The provider_location_update event listener is the most critical part. It receives location data from a provider's phone and does two things simultaneously: broadcasts it to the user's browser (via WebSockets) and updates the MongoDB database (for geospatial queries).

// Core Socket Logic
io.on('connection', (socket) => {
    socket.on('provider_location_update', async ({ roomId, userId, location }) => {
        // 1. Direct Broadcast (User sees car move)
        if (roomId) io.to(roomId).emit('track_provider', location);

        // 2. DB Update (For "Search Nearby" feature)
        if (userId) {
            await User.findByIdAndUpdate(userId, {
                liveLocation: { type: 'Point', coordinates: [location.lng, location.lat] },
                isOnline: true
            });
        }
    });
});

Google Maps Wrapper

client/src/components/GoogleMaps/MapComponent.jsx

A reusable wrapper around the Google Maps Javascript API. It handles script loading, theme switching (Dark/Light/Satellite), and geolocation.

Custom Themes

We use a custom JSON array darkMapStyle to skin the map. This removes standard POIs that clutter the view and gives it the "Uber Night Mode" look with dark roads and muted text.

Performance

Uses useMemo for mapOptions to prevent the map from re-rendering unnecessarily when parent components update. This is crucial for smooth 60fps animations.

// Map Theme Logic
const mapOptions = useMemo(() => ({
    styles: isDark ? customDarkStyle || darkMapStyle : [],
    disableDefaultUI: true, // We hide Google's buttons
    zoomControl: false,     // And build our own custom UI
}), [isDark]);

Live Chat System

A self-contained React functional component. It doesn't rely on global state for messages; it keeps its own local history messages array and appends to it when the socket fires receive_message.
