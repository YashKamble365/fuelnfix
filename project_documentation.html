1. Introduction & Problem Statement

1.1 Background

Vehicle breakdowns are unpredictable events that cause significant stress and inconvenience. Traditional roadside assistance services suffer from:

  • Long wait times: Average response time exceeds 45 minutes in urban areas
  • Lack of transparency: No visibility into arrival time or pricing
  • Trust issues: Difficulty verifying mechanic credentials
  • Payment friction: Cash-only transactions are risky and inconvenient

1.2 Solution Overview

FuelNFix is a comprehensive on-demand roadside assistance platform that connects stranded motorists with verified local service providers. Often described as the "Uber for Mechanics", it leverages real-time GPS tracking, intelligent matching algorithms, and secure digital payments to deliver:

  • Average response time under 15 minutes
  • Complete price transparency before booking
  • Verified and rated service providers
  • Cashless, secure transactions

1.3 Scope

The platform supports three service categories:

  1. Mechanic Services: Flat Tire, Dead Battery, Towing, Key Lockout, Engine Trouble
  2. Fuel Delivery: Petrol and Diesel delivery to stranded vehicles
  3. EV Support: Mobile EV Charging, Flatbed Towing, Battery Jumpstart, Cable Unlock

8. Core Algorithm & Logic

8.1 Provider Matching Algorithm

The search algorithm uses MongoDB's geospatial capabilities to find optimal providers.

// File: server/controllers/requestController.js

exports.searchProviders = async (req, res) => {
    const { serviceName, originLat, originLng, category } = req.body;
    
    // Build geospatial query
    const query = {
        role: 'provider',
        isVerified: true,    // Only verified providers
        isOnline: true,      // Only active providers
        providerCategory: category,
        location: {
            $near: {
                $geometry: { 
                    type: "Point", 
                    coordinates: [originLng, originLat] 
                },
                $maxDistance: 50000  // 50km radius
            }
        },
        // Must offer the requested service
        services: {
            $elemMatch: { 
                name: { $regex: new RegExp(`^${serviceName}$`, 'i') }, 
                active: true 
            }
        }
    };
    
    // Find providers (automatically sorted by distance due to $near)
    const providers = await User.find(query).limit(50);
    
    // Calculate pricing for each provider
    const results = providers.map(provider => {
        const distance = calculateHaversineDistance(
            originLat, originLng,
            provider.location.coordinates[1],
            provider.location.coordinates[0]
        );
        
        const distanceFee = distance * pricePerKm;
        const totalEstimate = baseFee + distanceFee;
        
        return {
            provider: { id, name, shopName, rating, location },
            distance: distance.toFixed(2),
            baseFee,
            distanceFee: distanceFee.toFixed(2),
            totalEstimate: Math.ceil(totalEstimate)
        };
    });
    
    res.json(results);
};

8.2 Haversine Distance Formula

// Calculate distance between two lat/lng points on Earth
const calculateDistance = (lat1, lon1, lat2, lon2) => {
    const R = 6371;  // Earth's radius in km
    const dLat = (lat2 - lat1) * (Math.PI / 180);
    const dLon = (lon2 - lon1) * (Math.PI / 180);
    
    const a = Math.sin(dLat/2) * Math.sin(dLat/2) +
              Math.cos(lat1 * Math.PI/180) * Math.cos(lat2 * Math.PI/180) *
              Math.sin(dLon/2) * Math.sin(dLon/2);
              
    const c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a));
    return R * c;  // Distance in km
};

8.3 Pricing Formula

Total Price = Base Fee + (Distance × Rate per km) + Material Cost

For Fuel Delivery: Total = Base Fee + (Distance × Rate) + (Litres × Price per Litre)

10. Payment Integration (Cashfree)

10.1 Payment Flow

  1. Provider clicks "Send Bill" → Bill items stored in Request
  2. Customer receives bill via Socket event
  3. Customer clicks "Pay" → Frontend calls /api/payment/create-order
  4. Server creates Cashfree order, returns payment_session_id
  5. Frontend opens Cashfree SDK with session ID
  6. User completes payment in Cashfree UI
  7. Cashfree redirects back with order status
  8. Frontend calls /api/request/confirm-payment
  9. Request status updated to "Completed"
  10. Provider notified via Socket

10.2 Create Order Implementation

// server/routes/paymentRoutes.js

router.post('/create-order', async (req, res) => {
    const { orderAmount, customerId, customerPhone, customerName } = req.body;
    
    const orderId = `ORDER_${Date.now()}`;
    
    const request = {
        order_amount: orderAmount,
        order_currency: "INR",
        order_id: orderId,
        customer_details: {
            customer_id: customerId,
            customer_phone: customerPhone,
            customer_name: customerName || "Customer"
        },
        order_meta: {
            return_url: "https://fuelnfix.com/return?order_id={order_id}"
        },
        order_note: "FuelnFix - Roadside Assistance"
    };
    
    const cashfree = new Cashfree(
        CFEnvironment.SANDBOX,  // Change to PRODUCTION for live
        process.env.CASHFREE_APP_ID,
        process.env.CASHFREE_SECRET_KEY
    );
    
    const response = await cashfree.PGCreateOrder(request);
    res.json(response.data);  // Contains payment_session_id
});

11. User Manual

11.1 Customer Guide

Step 1: Registration

  1. Open FuelNFix and click "Get Started"
  2. Sign in with your Google account
  3. Select "I need help" (Customer role)
  4. Add your vehicle details (optional but recommended)
  5. Allow location permission when prompted

Step 2: Requesting Service

  1. From the Dashboard, click "Request Help"
  2. Select your vehicle from the list (or add a new one)
  3. Choose the issue type (e.g., "Flat Tire")
  4. Confirm your location on the map
  5. Review the estimated price and available providers
  6. Select a provider and click "Confirm Request"
Tip: You can select multiple services if needed. The price will be calculated for all services combined.

Step 3: Tracking & Verification

  1. Wait for a provider to accept (you'll hear a notification sound)
  2. Watch the provider approach on the live map
  3. When they arrive, you'll see a 4-digit OTP on your screen
  4. Only share this OTP when you physically see the provider
  5. The job begins once OTP is verified
Security: Never share your OTP over phone or before verifying the provider's identity.

Step 4: Payment & Receipt

  1. After service completion, you'll receive a digital bill
  2. Review the itemized charges (service fee + materials)
  3. Click "Pay Securely" to open the payment gateway
  4. Complete payment using UPI, Card, or Netbanking
  5. Download your receipt for records/insurance claims

11.2 Provider Guide

Onboarding

  1. Register with Google and select "I provide services"
  2. Enter your shop name and address
  3. Select your category (Mechanic, Fuel Delivery, or EV Support)
  4. Choose the services you offer from the list
  5. Upload your shop photo and business proof document
  6. Set your shop location on the map
  7. Submit for verification (wait 24-48 hours)

Daily Operations

  1. Go Online: Toggle the switch in Provider Dashboard
  2. Receive Request: A popup shows distance, service type, and estimated earnings
  3. 60-Second Timer: Accept or reject within the time limit
  4. Navigate: Click the navigation button to open Google Maps
  5. Verify OTP: Ask the customer for their 4-digit code
  6. Complete Work: Perform the service
  7. Generate Bill: Add any material costs and send the bill
  8. Receive Payment: Customer pays through the app

11.3 Admin Guide

Accessing Admin Dashboard

Navigate to /admin. Only users with role: 'admin' can access.

Provider Verification

  1. Go to "Pending Verification" tab
  2. Click on a provider to expand details
  3. View uploaded documents (shop photo, business proof)
  4. Click "Approve" to activate or "Reject" to delete

Managing Service Rates

  1. Go to "Pricing" tab
  2. Edit base price, per-km rate, or per-litre rate
  3. Changes apply immediately to new searches

Sending Announcements

  1. Go to "Broadcasts" tab
  2. Enter title and message
  3. Select target audience (All, Users only, Providers only)
  4. Click "Send Broadcast"
  5. Message is immediately pushed to all connected clients

13. Security Considerations

13.1 Authentication

  • Firebase handles identity (Google OAuth, Phone Auth)
  • Backend verifies Firebase ID tokens on every authenticated request
  • No passwords stored locally (delegated to Firebase)

13.2 Data Protection

  • All API communication over HTTPS
  • MongoDB Atlas encrypts data at rest
  • Sensitive keys stored in environment variables, never in code
  • .gitignore excludes .env, serviceAccountKey.json

13.3 OTP Verification

  • 4-digit OTP generated server-side on request acceptance
  • OTP only visible to customer, provider must ask verbally
  • Job cannot start without OTP verification
  • Prevents unauthorized service starts and fraud

13.4 Location Privacy

  • Live location only broadcast to request-specific "room"
  • Only the customer and assigned provider can see tracking
  • Location stops broadcasting when job completes

14. FAQ & Troubleshooting

Q: Provider search returns empty results

A: Check that providers have: (1) isVerified: true, (2) isOnline: true, (3) Valid location.coordinates, (4) The requested service in their services array with active: true.

Q: CORS error in browser console

A: Ensure your frontend URL is added to the CORS whitelist in server/index.js.

Q: Google Maps not loading

A: Verify your Google Maps API key has Maps JavaScript API enabled and your domain is whitelisted in Google Cloud Console.

Q: Socket.IO not connecting

A: Check that both frontend and backend are using compatible Socket.IO versions (both v4.x). Verify the socket URL matches your backend URL.

Q: Payment fails with "Invalid credentials"

A: Ensure CASHFREE_APP_ID and CASHFREE_SECRET_KEY are correct. For testing, use Sandbox credentials.

Q: "Can't extract geo keys" MongoDB error

A: Location coordinates must be in GeoJSON format: [longitude, latitude] (not lat, lng). Ensure they are Numbers, not Strings.
